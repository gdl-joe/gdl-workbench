Release Notes

Version 1.6.0

23. Februar 2026 – Codeschmiede Update
  • Neue Seite: Codeschmiede (GDL Code Forge):
    • Interaktives Werkzeug zur Generierung von GDL-Code-Bausteinen aus Eingabemasken.
    • Drei Modi: Manuell, Aus Parametern, → Parameter (Variablen direkt als Parameter speichern).
    • Mehrsprachige Navigation: i18n-Schlüssel nav_codeschmiede in allen 4 Sprachen ergänzt.
  • Feature: Befehlsbaum-Suche:
    • Suchfeld im linken Panel filtert Befehle live nach Name, Kategorie oder Unterkategorie.
    • "Keine Befehle gefunden"-Anzeige bei leerem Suchergebnis.
  • Feature: Snippets in MySQL-Datenbank:
    • Snippets (gespeicherte Parametersätze) werden in der Datenbank gespeichert statt in localStorage.
    • Neue Tabelle codeschmiede_snippets (Migration 003).
    • Neuer PHP-Controller CodeschmiedeController mit GET / POST / DELETE Endpunkten.
    • Snippets im Befehlsbaum mit Datum-Tooltip und Löschen-Button.
  • Neuer GDL-Befehl: POLY2_B – Polygon mit Bogenkanten:
    • Beliebiges n-Eck mit gemischten geraden und gebogenen Kanten.
    • Stichhöhe z_poly[i] = 0 → gerade Kante; ≠ 0 → Bogenkante.
    • Array-Parametertypen length_array / integer_array eingeführt.
    • Funktion resolveParamType() für transparente Array-Erkennung im Formular.
    • Modus "→ Parameter" legt Array-Parameter mit korrektem array_type und array_first_dim an.
  • Feature: Koordinaten-Tabelle (coord-table):
    • Neuer Parametertyp coord-table: interaktive N×3-Tabelle für x/y/z-Koordinaten.
    • Zeilen passen sich dynamisch an die Punktanzahl (int_poly_n) an.
    • Spaltenköpfe aktualisieren sich live mit den Array-Variablennamen.
    • Sitzungszustand (State) und Snippets speichern die Tabellenwerte als JSON.
    • In Modus "Aus Parametern" ausgeblendet (Arrays kommen aus vorhandenen Parametern).
  • Code-Optimierung POLY2_B-Bogen-Script:
    • Zwei Schleifen + GOSUB 12 + Subroutine zu einer einzigen Schleife zusammengefasst.
    • Intermediate Arrays (poly_dx[], poly_dy[], r2[], xm[], ym[]) durch skalare Temp-Variablen ersetzt.
    • Nur noch arc_xm[], arc_ym[], arc_ang[] als Arrays notwendig.
    • DIM-Deklaration für die Koordinaten-Arrays automatisch generiert.
